- Create a Player class that owns player movement, jumping, gravity, vertical speed, and screen-boundary handling.
- The UI form should forward keyboard input and call Player methods rather than changing movement variables directly.
- Extract reusable enemy and moving platform classes
- Refactor duplicate movement logic into reusable classes such as Enemy and MovingPlatform. Each object should manage its own speed, direction, and update behavior.

Deliverables for this slice
- Player class with attributes/properties and methods such as MoveLeft, MoveRight, StartJump, AplyMechanics
- Old player logic moved out of frmMain as much as possible
- Working slice showing left/right movement, jumping, gravity, and falling
- At least one reusable Enemy class used for both enemies
- At least one reusable MovingPlatform class used for both moving platforms
- Working slice showing enemy motion and platform motion with correct reversals at limits
